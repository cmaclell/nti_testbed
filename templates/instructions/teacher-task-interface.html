Task Interface

Overview

For this experiment, you will be teaching an AI to control a robot in a simulated office environment. The simulated robot will be able to sense its environment by emitting lasers, which it can use to determinue where walls are and detect objects it can interact with in its environment. Additionally, the robot has an arm that it can use to pickup and put down objects in the environment.

Interface Example

Here is an example of someone using the task interface that you will be guiding the AI student to use.

Possible Actions

The interface is for controlling a simple simulated robot that can sense and act in its environment. There are three possible actions in this interface:

  • Clicking on the screen sets a waypoints for the robot to move to. As the robot moves it actively senses its environment and updates its model, as reflected by the white dots, which correspond to locations where its sensors detected a wall or obstacle, and the colored shapes, which correspond to objects in the robot has detected in the environment.
  • Pressing the "Action 1" button while over an object (the colored shapes) makes the robot pickup the object and put it in its inventory. Only one object can be held at a time.
  • Pressing the "Action 2" button will drop an object that is currently in the robot's inventory.

Avoiding Obstacles

The robot is unable to move through obstacles. If it encounters an obstacle it will do its best to continue following its waypoints, but if it gets stuck then it will stop moving and its waypoints will reset. For more efficient movement, try not to set waypoints that cause the robots path to encounter any obstacles
